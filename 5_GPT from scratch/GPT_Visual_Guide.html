Every word-piece gets an integer ID, and that ID maps to a row in the embedding table. Similar words end up in nearby regions of the 768-dimensional space — not because we told the model to, but because predicting the next token forces it to group similar contexts together.

Sliding Window — One Book Becomes Millions of Training Examples

The training task is always the same: given the previous N tokens, predict the next one. With stride=128 and 1.89M training tokens, the model sees hundreds of thousands of overlapping (context → next) pairs from the same books — each one a slightly different view of the same text.

02 · Attention — Learning What to Focus On

Query · Key · Value — Attention as a Soft Database Lookup

Q, K, V are three different learned projections of the same input. Scoring Q against all K's simultaneously means every token can consider every other token in one step — impossible with RNNs, which process sequentially and lose context from many tokens ago.

Causal Mask — The Triangle That Prevents Cheating

The upper-right triangle is set to −∞ before softmax. Softmax(−∞) = 0, so those positions contribute nothing. This one mask enforces causality: token at t=1 can see t=0 and t=1, never t=2 or t=3. Training and generation use the exact same forward pass.

Multi-Head Attention — 12 Parallel Perspectives

Each of 12 heads runs independent attention on a 64-dim slice. Because 12 × 64 = 768, the output shape is identical to the input. Each head can specialize in different relationships (syntax, coreference, position) without being instructed to — this emerges from training alone.

Why multiple heads beat one wide head: One 768-dim head produces a single weighted blend of all tokens. Twelve 64-dim heads produce twelve independent blends that are then combined — giving the model multiple simultaneous views of the same sequence.
03 · GPT Model — Assembling 124 Million Parameters

Architecture — Shape Journey: (B, T) → (B, T, 50,257)

The 768-dimensional channel is the model's working memory width. Its shape never changes through 12 blocks. What changes is the information encoded in those 768 numbers — early blocks handle syntax and token patterns, later blocks handle abstract meaning and long-range context.

Inside One Transformer Block — Two Sub-Layers + Two Skip Connections

Skip connections (green + circles) add the sub-layer input directly to the sub-layer output. Gradients can flow backward through the addition, bypassing the sub-layers entirely. This is what makes 12-layer networks trainable — without skip connections, gradients vanish before reaching early layers.

Pre-norm (LayerNorm before each sub-layer): Each sub-layer always receives well-scaled inputs. Without it, activations drift after several layers and training becomes unstable.

FeedForward Network + Where the 124M Parameters Live

The 4× expansion (768→3,072) gives each token headroom to store complex associations. GELU allows small negative values near zero — ReLU zeroes all negatives, killing neurons permanently during training ("dead neurons").

04 · Training — From Random Weights to Learned Language

The 4-Step Training Loop — Repeated Millions of Times

The backward pass (③) computes how every weight contributed to the error — using the chain rule across all 12 layers and 124M parameters. PyTorch's autograd handles this automatically. The optimizer step (④) then nudges each weight a tiny amount in the direction that reduces loss.

Loss Curve + Five Trainer Generations

V0 — Sanity check
Overfit one batch to loss < 0.01. Proves the loop works before using real data. If this fails, something is broken.
V2 — ~4× faster
FP16 halves memory bandwidth. Flash Attention never materialises T×T matrix. torch.compile fuses GPU kernels.
V3 — Stability
Gradient clip (‖g‖ ≤ 1.0) stops spikes. Cosine LR: warm-up then smooth decay. Grad accumulation simulates large batch.
V4 — Multi-GPU (DDP)
Full model on each GPU, unique data per GPU. AllReduce averages gradients after each step. Near-linear speedup. Launch: torchrun.
Gradient accumulation: Run 32 micro-batches of 4 samples each. Accumulate (sum) the gradients. Then do one optimizer step. Mathematically identical to a real batch of 128 — because gradients are additive. Lets a consumer GPU simulate a cluster-scale batch.
05 · Inference — Controlling What the Trained Model Says

Temperature — The Certainty vs Creativity Dial

Temperature divides logits before softmax: probs = softmax(logits / T). The model weights are frozen — only how you sample from them changes.

Temperature 1.00

Real outputs from the notebook — same weights, same prompt "I have a dream":
T=0 (greedy): "…of being a doctor. I have a dream of being a doctor. I have a dream…"
T=0.8 + top-k/p: "…that I will be able to live in a world where I will be able to pursue my passion…"

Top-k and Top-p — Removing Low-Quality Candidates

Best in practice: combine both. k=50 ensures you never sample from truly garbage tokens. p=0.9 then further narrows to the likely nucleus. T=0.8 controls the final randomness. This is the ChatGPT / Claude default.

Beam Search — Track 3 Paths, Keep the Best

Greedy picks the best token at each step. Beam search tracks B=3 full candidates simultaneously and selects the best complete sequence. It always scores better by log-probability — yet still produces repetitive text. This reveals a fundamental gap between the model's objective (predict next token) and human judgment of quality.

Questions to Test Your Intuition
Data

The model is trained on Harry Potter and never receives any instruction like "learn English grammar." How does it learn grammar, punctuation, and style as a side-effect of predicting the next token?

Grammar is instrumentally useful: predicting "the dog runs" scores lower loss than "the dog run". The model discovers grammatically valid continuations are statistically more likely in the training data — grammar emerges as a compression strategy, never explicitly taught.
Attention

In the causal mask, t=0 can only attend to itself (weight = 1.00). t=3 splits attention across all four tokens. Does more context always improve predictions — and what breaks if you reverse the mask?

More context usually helps but not always — early tokens can overfit to spurious past correlations. Reversing the mask lets every token "see the future" during training, but future tokens don't exist at generation time. The train/inference mismatch makes autoregressive generation impossible.
Attention

Heads specialise in different relationships (syntax, coreference) without being told to. What mechanism in training causes this specialisation to emerge spontaneously?

Random initialisation seeds each head at a slightly different starting point. Gradient descent amplifies whatever a head is already slightly better at. Once a head captures a pattern, other heads face less gradient pressure to cover it — a feedback loop that drives diversity without any explicit instruction.
Architecture

The shape stays (B, T, 768) through all 12 blocks. If the shape doesn't change, what is actually different about the 768 numbers after block 12 versus after block 1?

Same shape, fundamentally different information. Block 1 output: mostly which token is at this position + immediate local context. Block 12 output: abstract concepts, topic, coreference chains, long-range dependencies. The 768 numbers represent a progressively richer semantic encoding of the sequence.
Architecture

Weight tying shares the embedding matrix between input and output. These operations seem to do opposite things (IDs → vectors vs vectors → logits). Why does sharing work and improve quality?

"dragon" and "wizard" are near each other in embedding space because they appear in similar contexts. The output head should also score them highly in similar contexts. The shared matrix enforces geometric consistency: the space used to understand words is the same space used to predict them.
Training

V0 overfits one batch to loss ≈ 0.01. The full 3-epoch run settles at loss ≈ 2.1. Why is 2.1 not considered a failure — and how would you distinguish capacity-limited from underfitting?

Random guessing from 50,257 tokens gives loss ≈ log(50,257) ≈ 10.8. Loss 2.1 means the model predicts the correct token ≈12× better than chance. To distinguish: if train ≈ val and both high → underfitting (need more capacity/data). If val ≫ train → overfitting (capacity exceeded the dataset).
Training

Gradient clipping caps the gradient norm at 1.0. What failure mode does it prevent — and why doesn't simply using a smaller learning rate solve the same problem?

A smaller LR slows ALL updates uniformly — including stable, well-calibrated ones. Gradient clipping only fires on catastrophic spikes (from anomalous batches or numerical edge cases) and leaves normal updates at full speed. It's a targeted guard against explosions, not a global slowdown.
Inference

Beam search finds the highest-probability sequence yet still produces repetitive text. What does this reveal about the gap between the training objective and what humans consider "good" text?

The loss never penalised repetition — it only rewarded next-token accuracy. "I have a dream of being a doctor" scored high at training time; repeating it scores high again. High probability ≠ high quality. Bridging this gap requires RLHF or DPO — separate objectives that directly reward human-judged quality.
